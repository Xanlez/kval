# Мероприятия (Django)

Веб-приложение для управления мероприятиями. Экзаменационный билет.

## Требования

- Python 3.10+
- Django 5+

## Установка

```bash
pip install -r requirements.txt
cp .env.example .env
python manage.py migrate
python manage.py runserver
```

Приложение: http://127.0.0.1:8000/

## Функциональность

- CRUD для мероприятий (список, создание, редактирование, удаление с подтверждением)
- Серверная валидация: пустое название, дата в прошлом, max_guests ≤ 0
- Кастомная страница 404
- Middleware для подсчёта запросов и кодов ответа (консоль + `metrics.log`)
- Настройки через `.env`
- Статические файлы через WhiteNoise (работает при `DEBUG=False`)
- Интеграционный тест главной страницы

## Модель `events`

| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER (PK) | Уникальный идентификатор |
| title | VARCHAR(200) | Название, не пустое |
| location | VARCHAR(150) | Место проведения |
| event_date | DATETIME | Дата и время, только в будущем |
| max_guests | INTEGER | Макс. гостей, > 0 |
| created_at | TIMESTAMP | Дата создания (авто) |
